SSL / TLS — How data in transit is protected

A compact infographic covering the purpose, core guarantees, handshake flow, and quick security notes for SSL/TLS.

Purpose

Protect data while it's moving across networks

Encryption, authentication, and integrity for connections (e.g., your browser ↔ server).

HTTPS / SMTPS / IMAPS / FTPS
TLS runs *inside* these application protocols
EncryptionConfidentiality — prevents eavesdropping.
AuthenticationVerify server (and optionally client) identity using certificates.
IntegrityDetect tampering using MACs or AEAD ciphers (e.g., AES-GCM).

TLS Handshake (simplified)

Client
(browser)
Server
(web server)
  1. ClientHello: client proposes TLS version, cipher suites, and sends a random.
  2. ServerHello: server picks parameters and sends its certificate.
  3. Key exchange: client verifies cert, establishes shared key (via ECDHE, RSA, etc.).
  4. Finished: both sides confirm keys and begin encrypted communication.
Note: modern TLS uses ephemeral key exchange (ECDHE) for forward secrecy and AEAD ciphers for combined encryption+integrity.
Certificate

Certificates (X.509) bind a domain name to a public key and are signed by a Certificate Authority (CA). Browsers trust a CA root list preinstalled in the OS/browser.

Quick security tips
  • Disable SSLv2/SSLv3 and TLS 1.0/1.1 — use TLS 1.2+ (ideally 1.3).
  • Prefer ECDHE for forward secrecy, AEAD ciphers (AES-GCM, ChaCha20-Poly1305).
  • Use HSTS, OCSP/CRL checks, and certificate transparency logs.

Common attack vectors

  • MITM (if certificates aren\'t validated) — attacker can intercept and impersonate.
  • Weak ciphers / old protocol — vulnerabilities like POODLE, BEAST, or RC4 weaknesses.
  • Misissued certificates — stolen/compromised CA keys or bad issuance.

Server checklist

  • Enable TLS 1.3 and 1.2 only
  • Use strong ephemeral ECDHE key exchange
  • Choose AEAD ciphers: AES-GCM or ChaCha20-Poly1305
  • Enable HSTS and implement secure cookies

Key terms

AEAD: Authenticated Encryption with Associated Data — encrypts and protects integrity.
ECDHE: Ephemeral Diffie-Hellman over elliptic curves — provides forward secrecy.
CA: Certificate Authority — signs certificates.
