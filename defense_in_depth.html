Defense in Depth — Layered Security Model

Perimeter → Network → Endpoint → Application → Data — multiple layers to slow attackers and reduce blast radius.

Concept

Defense in Depth (DiD)

1
Perimeter — Firewalls, IDS/IPS, VPNs, DDoS protection
2
Network — Segmentation, NAC, VLANs, Secure routing
3
Endpoint — EDR, patching, device hardening, MDM
4
Application — Secure SDLC, WAF, input validation, SAST/DAST
5
Data — Encryption (at rest & in transit), DLP, access control

Why layers matter
  • Reduces single points of failure
  • Provides detection and containment at multiple stages
  • Improves resilience and incident response time

Practical implementation tips

  • Use network segmentation to limit lateral movement.
  • Enforce least privilege for users and services.
  • Instrument logs and centralize SIEM for cross-layer detection.
  • Encrypt sensitive data and rotate keys securely.

Common risks

  • Overreliance on perimeter controls
  • Poor patch management at endpoints
  • Insufficient access controls and key management

Quick checklist

Inventory → Segment → Patch → Monitor → Encrypt
